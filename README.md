# 🏥 Hospital Management System - SQL Data Analysis Project

## 📖 Overview
The Hospital Management System is a SQL-based data analytics project that demonstrates how healthcare data can be managed and analyzed using SQL. The project focuses on patients, doctors, departments, and billing information to generate business insights and support data-driven decision-making.

---

## 🎯 Objective
- Design a relational hospital database.
- Analyze patient, doctor, and billing data using SQL.
- Generate reports and business insights for hospital management.
- Demonstrate SQL concepts used in real-world healthcare analytics.

---

## 🛠️ Tech Stack
- SQL (MySQL / Oracle SQL)
- MySQL Workbench / Oracle SQL Developer
- Power BI
- Python (Pandas)
- Microsoft Excel
- GitHub

---

### Doctors

## 🔗 Database Relationships

## 📈 Business Questions Solved

- Display patient, doctor, and billing details.
- Find patients with bills greater than ₹30,000.
- Calculate department-wise revenue.
- Calculate average bill amount by department.
- Identify the doctor treating the highest number of patients.
- Find the highest, lowest, and average bill amount.
- Categorize bills using CASE statements.
- Display the Top 3 highest patient bills.
- Rank patients by bill amount.
- Calculate revenue generated by each doctor.
- Count patients by gender and department.

---

## 💡 Key Insights

- Cardiology generated the highest revenue.
- Dr. Sharma treated the highest number of patients.
- High-value bills contributed significantly to overall revenue.
- Most patients belonged to the 25–45 age group.
- Department-wise revenue analysis supports better budget planning.
- Doctor workload analysis helps optimize resource allocation.

---

## 🚀 Project Highlights

- Designed a relational SQL database.
- Performed data analysis using SQL.
- Applied Joins, Aggregate Functions, CASE Statements, Subqueries, and Window Functions.
- Generated business reports and actionable insights.
- Built a Power BI dashboard for interactive visualization.
- Used Python (Pandas) for data cleaning and preprocessing.
